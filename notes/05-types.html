Types

In computer computer programming languages, all values have a type.

The type defines:

  • the set of values of that type,

  • the literals for those values (how we write the values in programs), and

  • the available operators and operations (what we can do with those values).

For integers:

  • the values/literals are …, $-3$, $-2$, $-1$, $0$, $1$, $2$, $3$, …,

  • and the operators are the four normal operations of everyday arithmetic plus the modulus and power operators ("%" and "**").

In python, we can query the type of any value:

>>> type(3)
<type 'int'>

>>> x = 3
>>> type(x)
<type 'int'>

>>> type(x+7)
<type 'int'>

The built-in python function type() returns the type of any value.

int is python’s internal name for the integer type.

More types:

Here, we introduce three more python types:

  • Booleans (bool)

  • strings (str)

  • floating-point numbers (float)

Booleans

Our first new type is the Boolean type.

There are lots of different integer values, about $2^64$ different values. That really is a lot.

For Booleans, there are only two values.

The there are just two literals; these are:

  • True

  • False

Note

Note the capital "T" and the capital "F".

As it happens — and as we will see below — we rarely use Boolean literals in computer programs.

As before, we can query the type of Boolean values too:

>>> type(True)
<type 'bool'>

>>> type(False)
<type 'bool'>

>>> x = True
>>> type(x)
<type 'bool'>

We see that python’s internal name for the Boolean type is bool.

An integer expression is an expression which evaluates to a integer value.

Similarly, a Boolean expression is an expression that evaluates to a Boolean value.

Here, we cover two important forms of Boolean expressions:

  1. comparison operators, and

  2. Boolean operators.

Comparison operators

Comparisons are expressions; they evaluate to Boolean values.

Examples 1:
3 < 5         # True.

x = 3         # Assignment statement (not a comparison)!

x < 5         # True.
x > 5         # False.

Just as integer expressions evaluate to integer values, Boolean expressions (including comparisons) evaluate to Boolean values.

Examples 2:
4 <= 4        # True.

a = 3         # Assignment statement (not a comparison)!
b = 4         # Assignment statement (not a comparison)!

a <= b        # True.
a >= b        # False.

a == b        # False.
a != b        # True.

There are six comparison operators in python:

a <  b        # True if value a is less than value b, False otherwise.
a <= b        # True if value a is less than or equal to value b, False otherwise.

a >  b        # True if value a is greater than value b, False otherwise.
a >= b        # True if value a is greater than or equal to value b, False otherwise.

a == b        # True if values a and b are equal, False otherwise.
a != b        # False if values a and b are equal, True otherwise.
Note

These are different:

x == 3        # This is a Boolean expression, it evaluates to either True or False.

x = 3         # This is an assignment statement.

In python, and in several other important programming languages:

  • a single equals sign is always an assignment statement, and

  • a double equals sign is always a comparison operator.

It is a common error for beginner and expert alike to mix these two up.

Boolean operators

A second way to construct Boolean expressions is via Boolean operators.

There are three Boolean operators: and, or and not:

              # Assume existing Boolean variables a and b.

a and b       # True if *both* a and b are True, False otherwise.
a or b        # True if *either* of a and b is True (or both), False otherwise.
not a         # True if a is False, False otherwise.

Examples I

Assume that integer variables $x$ and $y$ are the lengths of the bottom and side (respectively) of an axis-aligned rectangle.

Below, in each case, we are looking for Boolean expression which will be True if the stated condition holds, and is False otherwise.

Is the rectangle also a square?
x == y       # This will evaluate to True if the rectangle happens to
             # be a square, and False otherwise.
Is the rectangle not a square?
x != y
Is the rectangle taller than it is wide?
x < y
Is the rectangle wider than it is tall?
y < x
Is the rectangle’s area less than 100?
x * y < 100
Is the rectangle’s area no more than 100:
x * y <= 100

Each of these evaluates to a Boolean value, either True or False, depending upon the lengths of the sides, as represented by the values of variables $x$ and $y$.

Examples II

Assume an existing integer variable month representing a month (in the range [1..12]).

Is the month February?
month == 2
Does the month have thirty days?
month == 4 or month == 6 or month == 9 or month == 11
Does the month have thirty days?
# NOT LIKE THIS!

month == 4 or 6 or 9 or 11    # WRONG!!!

# "or" is a Boolean operator.
#
# The values on the left and right must be Boolean values; here,
# they are integers (so this is wrong).

Examples III

Assume an integer variable year is a year (e.g. $2005$).

Leap years:

  • years which are divisible by $400$ are leap years,

  • otherwise, all years which are divisible by $4$ are leap years, unless they are also divisible by $100$.

(Complicated, isn’t it?)

Is year a leap year?
(divisible by 400) or (divisible by 4 but not by 100)

We can check divisibility by checking whether the remainder after division by a number is $0$.

For example:

  • $21$ is divisible by $7$ because the remainder after dividing $21$ by $7$ is $0$, and

  • $22$ is not divisible by $7$ because the remainder after dividing $22$ by $7$ is not $0$ (it’s $1$).

For leap years, we can check — for example — whether a number is divisible by $400$ as follows:

year % 400 == 0
Examples
1998 % 400 == 0      # False (the remainder is 398).
1999 % 400 == 0      # False (the remainder is 399).
2000 % 400 == 0      # True  (the remainder is 0).
2001 % 400 == 0      # False (the remainder is 1).
2002 % 400 == 0      # False (the remainder is 2).
Is year a leap year?

To test whether a year is a leap year, we need to translate the definition above into python:

# A year is leap year if it is:
#
#    (divisible by 400) or (divisible by 4 but not by 100)
#     |              |      |            |     |        |
#    /              /      /            /     /        /
#   /              /      /            /     /        /
#  /              /      /            /     /         |
# |              /      /            /     /          |
# |             |      /            /     /           |
# |             |     |            /     /            |
# |             |     |           |     |             |
  year % 400 == 0 or (year % 4 == 0 and year % 100 != 0)

Examples IV

Degree classifications at DCU are as follows:

  1. a mark of 70% or more is a first,

  2. a mark of 60% to 69% is a 2.1,

  3. a mark of 50% to 59% is a 2.2,

  4. a mark of 40% to 49% is a 3rd, and

  5. any lower mark is a fail.

Assume a python script which starts like this:

mark = int(input())

For each of the five classes above, write a Boolean expression which is True if the mark falls into the into the indicated range.

# 1. First.
70 <= mark

# 2. 2.1.
60 <= mark and mark < 70

# 3. 2.2.
50 <= mark and mark < 60

# 4. 3rd.
40 <= mark and mark < 50

# 5. Fail.
mark < 40

Observation

There are two Boolean values, and two Boolean literals, True and False.

In practice, however, it is uncommon for those Boolean literals to appear in Python scripts.

Rather, Python scripts more commonly include Boolean expressions which — when evaluated — evaluate to Boolean values.

Strings

It is common for computer programs to represent textual data.

The python type for textual data is string.

The name "string" may perhaps appear unusual, but it is the name which is widely used for textual data types across many programming languages.

A string is a sequence of characters.

The internal python name for the string type is str.

String literals:
"Hello world!"
"January"
"Tuesday"
"Python"
"Computer Programming I"
"Nobody expects the Spanish Inquisition!"

String literals are enclosed in either double quotes (") or single quotes (').

Usually, you should prefer double quotes (unless the textual data itself contains double quotes).

But be consistent: choose one, and stick to it.

Examples
s = "Who's afraid of Virginia Woolf?"

t = 'And the band played "Waltzing Matilda".'

What can we do with strings?

Read a string from standard input:
s = input()
t = input()
We can store textual data values in variables:
module_code = "CA116"
module_title = "Computer Programming I"
We can print strings:
print("Hello!")
print("Welcome to", module_title)
Output:
Hello!
Welcome to Computer Programming I
We can ask how long they are:
len("Hello")                 # 5
len("Hello World!")          # 12
len(module_title)            # 22

len() is a built-in function in python; we use it for querying how long something is.

We can use comparisons:
month = "January"            # Assignment (not a comparison)!

month == "January"           # True
month == "February"          # False
month != "February"          # True

"abc" < "def"                # True (alphanumerical ordering)
"def" < "abc"                # False (alphanumerical ordering)
"100" < "9"                  # True (alphanumerical ordering!)
We can ask what type they are:
type("Hello")                 # <type 'str'>
type("Hello") == str          # True
type(3) == str                # False
Note

We will return to talk more about what we can do with strings later; but that’s enough for now.

Floating-point numbers

Floating-point numbers are python’s approximation to decimal-point numbers in arithmetic.

Perhaps surprisingly, floating-point numbers come up only in very particular types of computational problem.

Literals
3.14          # Floating-point number.
3.0           # Floating-point number.
1.0           # Floating-point number.
0.0           # Floating-point number.
1             # Integer.
123           # Integer.
Example
#!/usr/bin/env python3

pi = 3.14
radius = int(input())

print "the area of this circle is", pi * radius * radius
Arithmetic operators

In a numeric expression:

  • if all of the terms are integers, then the result will be an integer,

  • but, if any one of the terms is a floating-point number, then the result will be a floating-point number.

So, above, because pi is a floating-point number, the entire expression evaluates to a floating-point number.

More about print

Example:
months = 12
print("there are", months, "months")    # "there are 12 months\n"

# (Here, "\n" represents a newline character.)

print:

  • accepts 0, 1 or more comma-separated arguments

  • converts each argument to a string

  • joins each argument together, separating them with a space

  • adds a new-line character to the end,

  • and writes the resulting text to standard output.

We can print integers, Booleans and strings, and print produces more or less the output you would expect.

Example
print(3 < 4)
# Prints True.

print(4 < 3)
# Prints False.
