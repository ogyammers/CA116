Statements

Thus far, we have seen two kinds of statement: assignments statements and print statements.

These notes cover if statements, a third kind of statement.

Conditional execution

Conditional execution means that we define a block of statements which may or may not be executed, depending upon some condition.

An if statements is used for conditional execution.

The simplest form of an if statement is:

if BOOLEAN_EXPRESSION:
   STATEMENT_1            # These statements constitute
   STATEMENT_2            # the "if block".
   ...                    #

FOLLOWING_STATEMENTS
...

BOOLEAN_EXPRESSION, above, is any Boolean expression.

When the python interpreter encounters an if statement, it evaluates the Boolean expression:

  • if the resulting value is True, then the statements within the if block are executed (from top to bottom), before moving on to any following statements;

  • otherwise, the rest of the if statement is skipped, and the python interpreter moves directly on to the following statements.

The Boolean expression is also known as the condition.

One way to view this as a flow chart…

             -------------
            /   BOOLEAN   \
            \ EXPRESSION  / ------
             -------------       |
                   |             |
                   | True        | False
                   |             |
            ---------------      |
            | if block    |      |
            |             |      |
            ---------------      |
                   |             |
                   |<-------------
                   v
            ---------------
            | FOLLOWING   |
            | STATEMENTS  |
            ---------------
Example Task

Assume that we represent a day as an integer, with $0$ representing Monday and $6$ representing Sunday.

Write a python script which reads a day and outputs the message weekend if that day is Saturday or Sunday (and nothing otherwise).

Example Solution
#!/usr/bin/env python3

day = int(input())

if day == 5 or day == 6:
   print("weekend")

print("done")

In this case, there is only one statement within if block.

The condition is a Boolean expression. It will evaluate to either True or False:

  • if it evaluates to True, then the print statement in the if block will be executed,

  • otherwise, if the Boolean expression evaluates to False, then the print statement in the if block will skipped.

In both cases, the interpreter then moves on to the final print statement.

Syntax and indentation

Syntax

Example
if day == 5 or day == 6:
   print("weekend")

print("done")

An if statement always begins with the keyword "if".

A trailing colon (":") must follow the condition; this is a just required of python.

Indentation

Example
if day == 5 or day == 6:
   print("weekend")

print("done")

In python, the extent of a block (such as the if block, here) is determined by indentation.

The first statement within the block is indented, and subsequent statements of equal (or, in some circumstances greater) indentation are also within the block.

In the example, there is only one statement within the if block.

Following statements are then unindented again (that is, indented to the same level as the original if), as for the final print statement, above.

Example 1 (two statements within the if block)
#!/usr/bin/env python3

day = int(input())

if day == 5 or day == 6:
   print("weekend")
   print("done")

# The two prints above are both indented, so they are both within the if block;
# they will either both be executed, or both be skipped.
Example 2 (one statement within the if block)
#!/usr/bin/env python3

day = int(input())

if day == 5 or day == 6:
   print "weekend"

print("done")

# Here, only one print is indented, the other is not.  So the first print is inside
# the if block (it will either be executed or skipped), but the second print is not
# indented, so it is a following statement, and will *always* be executed.
Example 3
#!/usr/bin/env python3

day = int(input())

if day == 5 or day == 6:
   print("weekend")
      print("done")

# This is a syntax error; the execution of this script will fail (IndentationError).
#
# The indentation on the last print statement neither continues the indentation level
# of the if block, nor resets the indentation level to that of the if.
Example 4
#!/usr/bin/env python3

day = int(input())

if day == 5 or day == 6:
   print "weekend"
  print "done"

# This is an indentation error too.
#
# The indentation on the last print statement neither continues the indentation level
# of the if block, nor resets the indentation level to that of the if.

Tabs vs spaces

Always use space characters for indentation:

  • use two, three or four space characters.

(With fewer than two characters, the indentation isn’t sufficiently obvious, and with more than four characters, the lines tend to end up indented too far to the right.)

Never use tab characters.

Using tab characters is correct, however it can lead to bugs which can be difficult to find.

Note

On TermCast, the nano editor is configured to replace tab characters with spaces.

Be consistent. Pick a number of spaces to use and stick to it.

Two cases

A second form of conditional execution is where we want to do one thing or something else.

For this case, we use an if statement with an else block:

if BOOLEAN_EXPRESSION:
   STATEMENT_1            # This is the if block.
   STATEMENT_2            #
   ...                    #
else:
   OTHER_STATEMENT_1      # This is the else block.
   OTHER_STATEMENT_2      #
   ...                    #

As before, the Boolean expression is evaluated:

  • if it is True, then the if block is evaluated (as before) and the else block is skipped,

  • otherwise, the if block is skipped and the else block is evaluated.

Note

A colon (":") must follow the word "else".

The indentation of the if line matches that of the else line.

The flow chart looks like this…

             -------------
            /   BOOLEAN   \
            \ EXPRESSION  / ------
             -------------       |
                   |             |
                   | True        | False
                   |             |
            ---------------    ---------------
            | if block    |    | else block  |
            |             |    |             |
            ---------------    ---------------
                   |             |
                   |<-------------
                   v
            ---------------
            | FOLLOWING   |
            | STATEMENTS  |
            ---------------
Example Task

Assume that we represent a day as an integer, with $0$ representing Monday and $6$ representing Sunday.

Write a python script which reads a day and outputs either the message weekend (if that day is Saturday or Sunday), or the message weekday.

Example Solution
#!/usr/bin/env python3

day = int(input())

if day == 5 or day == 6:
   print("weekend")
else:
   print("weekday")

print("done")

If day is $5$ or $6$, then the condition evaluates to True, and we output weekend; otherwise we output weekday.

Either the if branch will be executed, or the else branch will be executed; but the python interpreter never executes both branches.

In both cases, the following print statement is then executed.

As before, the extent of the if and else blocks are both determined by indentation.

The if and the else must have the same amount of indentation, as follows:

if day == 5 or day == 6:
   print "weekend"
else:                      # Correct.
   print "weekday"

Not like this:

if day == 5 or day == 6:
   print "weekend"
   else:                   # Indentation error!
   print "weekday"

And not like this:

if day == 5 or day == 6:
   print "weekend"
 else:                     # Indentation error!
   print "weekday"

This is correct, but strongly discouraged:

if day == 5 or day == 6:
  print "weekend"          # Yuck.  These two do not have to match up,
else:                      # but you should nevertheless always take a
    print "weekday"        # consistent approach to indentation.

Multiple cases

Finally, it can happen that we must execute exactly one of some number of blocks (more than two).

We could do this by nesting if statements.

Here is an example.

Example Task

Assume, again, that we represent a day as an integer, with $0$ representing Monday and $6$ representing Sunday.

Write a python script which reads a day and outputs the name of the corresponding day (Monday, Tuesday, and so on).

Poor solution (do not copy this!)
#!/usr/bin/env python3

day = int(input())

if day == 0:
   print "Monday"
else:
   if day == 1:
      print "Tuesday"
   else:
      if day == 2:
         print "Wednesday"
      else:
         if day == 3:
            print "Thursday"
         else:
            if day == 4:
               print "Friday"
            else:
               if day == 5:
                  print "Saturday"
               else:
                  print "Sunday"
Yuck!

This is ugly; it is hard to get syntactically correct because of the complex indentation, and hard to maintain.

Moreover, the symmetry of the solution is not visually apparent.

However, this is a common case, and python provides a special syntax for such cases: the elif block.

Here’s a better solution.

Better solution
#!/usr/bin/env python3

day = int(input())

if day == 0:
   print "Monday"
elif day == 1:
   print "Tuesday"
elif day == 2:
   print "Wednesday"
elif day == 3:
   print "Thursday"
elif day == 4:
   print "Friday"
elif day == 5:
   print "Saturday"
else:
   print "Sunday"

There is a condition (a Boolean expression) in the if part, and in each elif part.

The python interpreter evaluates each condition in turn:

  • if the condition evaluates to True, then it executes the corresponding branch, and skips the rest of the if statement,

  • otherwise, if the condition evaluates to False, then it moves on to the next elif branch.

If all of the conditions evaluate to False, then the else branch is executed (if present).

The else branch is optional.

Colons

In all of the above, observe that lines beginning if, elif and else are all terminated with a colon.

These colons are always required: the python interpreter will fail with a syntax error if you omit one of these colon.

This is a fundamental (and recurring) detail of python’s syntax.

Examples

Examples I

Assume that integer variables $x$ and $y$ are the lengths of the bottom and side (respectively) of an axis-aligned rectangle.

Is the rectangle also a square?
if x == y:
   print("square")
else:
   print("oblong")
Is the rectangle not a square?
if x != y:
   print("oblong")
else:
   print("square")
Is the rectangle taller than it is wide?
if x < y:
   print("sky scraper")
else:
   print("factory")
Is the rectangle wider than it is tall?
if y < x:
   print("factory")
else:
   print("sky scraper")
Is the rectangle’s area less than 100?
if x * y < 100:
   print("small")
else:
   print("large")
Is the rectangle’s area no more than 100:
if x * y <= 100:
   print("small")
else:
   print("large")

Examples II

Assume an existing integer variable month representing a month (in the range [1..12]).

How many days are in the month?
if month == 2:
   print(28)
elif month == 4 or month == 6 or month == 9 or month == 11:
   print(30)
else:
   print(31)

Examples III

Assume an integer variable year is a year (e.g. $2005$).

How many days in the year?
if year % 400 == 0 or (year % 4 == 0 and year % 100 != 0):
   print(365)
else:
   print(366)

Examples IV

Degree classifications at DCU are as follows:

  1. a mark of 70% or more is a first,

  2. a mark of 60% to 69% is a 2.1,

  3. a mark of 50% to 59% is a 2.2,

  4. a mark of 40% to 49% is a third, and

  5. any lower mark is a fail.

Assume a python script which starts like this:

mark = int(input())

Write a python script which writes the degree classification to standard output.

if 70 <= mark:
   print("first")
elif 60 <= mark:  # X
   print("2.1")
elif 50 <= mark:
   print("2.2")
elif 40 <= mark:
   print("third")
else:
   print("fail")

Above, at the test marked "X", observe that we already know that the make is less than $70$ (because if the mark was $70$ or more, then the if branch above would have been executed instead).

Similar considerations apply to each of the subsequent elif conditions (and the else branch).
